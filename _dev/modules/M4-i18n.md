# M4: i18n & Yerelleştirme

**Sorumluluk:** Çok dilli altyapıyı sağlamak — next-intl routing/middleware, çeviri içerikleri ve RTL desteği.
**Bağımlılık:** Yok (taban katman). M2/M3/M5'teki tüm görünür metin bu modülden gelir.
**Sınır:** Yerelleştirme mekaniği ve çeviri içerikleri. Metnin görsel sunumu M2/M3'tedir.

---

## Feature'lar

### F4.1: next-intl routing & middleware → Faz —

**Açıklama:** `src/i18n/routing.ts` (`locales: [tr,en,ar,de,es]`, `defaultLocale: tr`, `localePrefix: as-needed`), `request.ts` (server config, mesaj dinamik import + fallback), `navigation.ts` (Link/redirect/usePathname/useRouter), `src/middleware.ts` (createMiddleware, matcher api/_next/dosya uzantılarını atlar).

**Kabul Kriterleri:**
- TR prefixsiz (`/`), diğerleri `/en` `/ar` `/de` `/es`.
- Bilinmeyen locale → defaultLocale fallback.
- Middleware matcher API ve internal rotaları atlar.

**Bağımlılık:** Yok

**Edge Case'ler:**
- `as-needed` prefix: TR root path'inin diğer dillerle çakışmaması.
- Tanınmayan locale segmenti güvenli fallback'lemeli.

---

### F4.2: Çeviri içerikleri → Faz —

**Açıklama:** `messages/{tr,en,ar,de,es}.json` — namespace yapılı (meta, nav, hero, how, sectors, bunker, forum, bunkerOs, chat, credibility, footer ...). `useTranslations()` (client) / `getTranslations()` (server) ile tüketilir.

**Kabul Kriterleri:**
- 5 dil aynı namespace anahtar setini taşır (eksik anahtar yok).
- AR/DE/ES gerçek çeviriler (placeholder/EN-aynalama değil — son commit'lerde tamamlandı).

**Bağımlılık:** F4.1

**Edge Case'ler:**
- Yeni metin eklerken 5 dilin de güncellenmesi gerekir (eksik anahtar runtime hatası/boşluk).
- **Revize hedefi:** Kopya kalitesi (özellikle TR) zayıf — yeniden yazım sırasında 5 dil senkron kalmalı; bkz. `docs/REVIZE-BACKLOG.md`.

---

### F4.3: RTL desteği (AR) → Faz —

**Açıklama:** AR locale'inde `dir="rtl"` (`[locale]/layout.tsx`), layout aynalama ve logical CSS prop'ları.

**Kabul Kriterleri:**
- AR'de `<html dir="rtl">` set edilir.
- Yerleşim AR'de mantıksal olarak aynalanır (okunabilirlik korunur).

**Bağımlılık:** F4.1, M1 (token), M2/M3 (layout)

**Edge Case'ler:**
- FlowScrim gradient yönü ve hizalamalar RTL'de metin tarafını doğru korumalı.
- Yeni bileşenlerde physical (left/right) yerine logical (start/end) prop tercih edilmeli.
